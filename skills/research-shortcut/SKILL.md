# Research Shortcut

## 工作流

### 1. 调用 tech-research 进行调研

对用户输入调用 `tech-research` 技能，严格执行`tech-research`规定的流程，不允许擅自发挥
执行不下去的时候要先报告，不允许自行决策

### 2. 逐子题深度检索

对每个子题执行搜索，收集以下六类信息（缺一不可）：

| 维度 | 要求 |
|------|------|
| 具体公司 | 公司全称、官网、总部、成立时间、核心定位 |
| 真实案例 | 标杆客户名称、应用场景、落地过程 |
| 技术实现细节 | 架构、算法、关键模块、数据流、集成方式 |
| 优势与效果 | 对比竞品的差异化能力 |
| 量化指标 | AHT 下降比例、效率提升百分比、ROI、用户规模等可验证数字 |
| 非常规核心技术 | 不常见但关键的技术手段（如专有模型微调、知识蒸馏、MCP 编排等） |

搜索时优先使用英文关键词获取一手资料，再用中文补充国内案例。

### 3. 按模板输出报告

- 若用户指定了输出模板，使用用户指定的模板
- 若用户未指定模板，**必须**读取并使用以下默认模板：
  `/Users/liuyuan/workspace/openclaw-home/workspace/skills/site-summarize/assets/summary-template.md`

按模板逐字段填充，不得遗漏任何字段。无法获取的字段标注"未公开/待补充"。

### 4. 质量自检

输出前检查：
- 每家公司至少有 1 个量化指标

- 所有事实须附来源链接
